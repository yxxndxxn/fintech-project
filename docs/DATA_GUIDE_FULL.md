# 데이터 연동 가이드

> 이 문서는 CSV·Excel 데이터를 받아 DSS 서비스에 연동하는 방법을 설명합니다.
> **Cursor에서 이 파일을 열어둔 채로 작업**하면 됩니다.

---

## 프로젝트 파일 구조

```
bank-dss/
  index.html          # 화면 (건드리지 않아도 됨)
  style.css           # 디자인 (건드리지 않아도 됨)
  script.js           # 계산 로직 (건드리지 않아도 됨)
  banks.js            # ★ 은행별 지점 데이터 — 여기를 채워야 함
  data/
    pairs.js          # 완료 — 10km 이내 대체 시설 데이터 (건드리지 않아도 됨)
    banks/            # 지역별 은행 위치 CSV (place_name, x, y 등)
    행정동_통합비율_취약점수_최종.xlsx  # 1번 데이터 — 행정동별 인구·고령자·취약점수
    지역별은행_더미데이터_전체.xlsx  # 2~4번 데이터 — 거래·연령·채널·λ·svc (은행별_더미데이터 시트)
  docs/
    DATA_GUIDE.md     # 이 문서
```

## 필요 데이터 ↔ banks.js 수정 위치 (핵심 매핑)

`banks.js`는 **4개 은행 데모 지점**(`hana`, `kb`, `shinhan`, `woori`)의 데이터를 담습니다.

| 필요 데이터 | banks.js 수정 위치 | `data/banks/*.csv`로 가능? |
|-------------|-------------------|---------------------------|
| 은행별 지점 정보 | `name`, `branch`, `pairKey` | 지점명·주소 확인만 가능 |
| 지역 인구·고령자 비율 | `s1.rows` [0], [1] | 가능 — `행정동_통합비율_취약점수_최종.xlsx` (지점→행정동 매핑 필요) |
| 방문 고객·창구 거래 | `s1.rows` [2], [5] + `s2.ages/channels` | 가능 — `지역별은행_더미데이터_전체.xlsx` `은행별_더미데이터` 시트 |
| 피크 시간당 고객 수 (λ) · ITM 처리시간 | `s3.lambda`, `s3.svc` | 가능 — 동일 xlsx에 **이미 계산됨** (별도 CSV 불필요) |
| 최근접 대체 점포 거리 | `s1.rows` [3] (`'—'` placeholder) | **이미 자동** — `data/pairs.js` + `pairKey` |

**중요:** `data/banks/서울_banks_최종.csv` 등 16개 CSV는 `place_name, address_name, road_address_name, x, y` **지점 위치 목록**입니다. 인구·거래 컬럼은 **없습니다**.

| 번호 | 데이터 | 연동 상태 | 담당 파일 |
|------|--------|-----------|-----------|
| 5 | 10km 이내 대체 시설·최근접 거리 | ✅ 완료 | `data/pairs.js` (자동 연동) |

| 2 | 지점별 방문객·거래·연령·채널 | ⏳ xlsx 연동 가능 | `banks.js` s1.rows, s2 ← `지역별은행_더미데이터_전체.xlsx` |

## banks.js — 어디를 고치는지

각 은행 블록(`hana`, `kb`, `shinhan`, `woori`) 안에서 수정 대상:

### 1. `s1.rows` — STEP 1 사전영향평가 표 (6개 행)

```js
s1:{ rows:[
  ['지역 인구','52,000','명'],           // [0] 인구 — 수정 대상
  ['65세 이상 고령자 비율','9.2','%'],   // [1] 고령자 — 수정 대상
  ['일평균 방문 고객','210','명'],        // [2] 방문객 — 수정 대상
  ['가장 가까운 대체 점포','—','km'],     // [3] 자동 계산 — 수정 불필요
  ['반경 500m 내 ATM/ITM','8','대'],     // [4] ATM 수 — 위치 CSV로 계산 가능
  ['월 창구 거래 건수','3,100','건']     // [5] 거래건수 — 수정 대상
], ...}
```

- **[3]번 행**은 `script.js`에서 `pairKey` → `PAIRS` 조회로 **화면 렌더 시 자동 덮어씀**. `'—'`는 placeholder입니다.
- **수정할 값은 `r[1]`(가운데 숫자)만** 바꾸면 됩니다.

### 2. `s2` — STEP 2 디지털 전환 적합성

- `ages`: [20~30대%, 40~50대%, 60대+%] — 합계 100
- `channels`: [모바일%, 자동화기기%, 대면창구%] — 합계 100

### 3. `s3` — STEP 3 대기행렬 입력

- `lambda`: 피크 시간당 고객 수 — xlsx `시간당 고객도착률 λ(ATM)` 또는 `λ(대표)` 사용
- `svc`: ITM 1건 처리 시간 (분) — xlsx `ATM 평균처리시간(분)` 사용

> `계산식_설명` 시트에 따르면 λ·μ·대기시간은 더미데이터 생성 시 **Sakasegawa 공식으로 이미 산출**되어 있으므로, 별도 `수요_예측.csv`나 λ·svc 전용 프롬프트가 필요 없습니다. (프롬프트 2에서 함께 반영)

### 4. `s1.score`, `s1.risks`, 판정 — 데이터 입력 후 계산

프롬프트 3 참고.

### 절대 건드리지 말 것

| 항목 | 이유 |
|------|------|
| `pairKey` | `data/pairs.js` 키와 1:1 매칭 필수 |
| `data/pairs.js` | 이미 완성 |
| `script.js`, `index.html`, `style.css` | 계산·화면 로직 |

### 데모 지점 ↔ pairKey ↔ 행정동 매핑

| banks.js ID | pairKey (변경 금지) | 주소(법정동) | xlsx 조회용 행정동 |
|-------------|---------------------|--------------|-------------------|
| hana | 하나은행 강남역금융센터지점 | 서울 서초구 서초동 | 서울특별시 / 서초구 / 서초동 |
| kb | KB국민은행 노원종합금융센터 | 서울 노원구 상계동 | 서울특별시 / 노원구 / 상계동 |
| shinhan | 신한은행 여의도중앙금융센터 | 서울 영등포구 여의도동 | 서울특별시 / 영등포구 / 여의도동 |
| woori | 우리은행 청담중앙지점 | 서울 강남구 청담동 | 서울특별시 / 강남구 / 청담동 |

> 주소는 `data/pairs.js`의 `주소` 필드 또는 `data/banks/서울_banks_최종.csv`의 `address_name`에서 확인할 수 있습니다.

## 행정동_통합비율_취약점수_최종.xlsx 구조

`data/행정동_통합비율_취약점수_최종.xlsx`는 취약점.pdf 방법론으로 산출된 **행정동 단위** 통합 데이터입니다.

### 시트 구성

| 시트명 | 용도 |
|--------|------|
| `행정동별_비율` | **메인** — 행정동별 인구·비율·취약점수 (banks.js 1번 데이터 소스) |
| `범례` | 컬럼·산출식 설명 |
| `시도별_학력보정계수` | 시도별 학력보정계수 참고값 |

### `행정동별_비율` 시트 주요 컬럼

| 컬럼 | banks.js 연동 |
|------|---------------|
| `시도`, `시군구`, `행정동` | 지점 주소 → 행정동 매핑 키 |
| `총인구수` | → `s1.rows` **'지역 인구'** |
| `고령인구비율` | → `s1.rows` **'65세 이상 고령자 비율'** |
| `기초취약도(Abase)`, `취약점수(A)` | 참고용 (현재 banks.js 미연동) |
| `장애인비율`, `수급권자비율`, `농어업인비율` | A 산출 중간값 (참고용) |

### 지점 → 행정동 조회 흐름

```
pairKey → pairs.js 주소 (또는 banks CSV address_name)
   → 법정동명 추출 (예: 서초동, 상계동)
   → xlsx '행정동별_비율'에서 시도+시군구+행정동 일치 행 조회
   → 총인구수, 고령인구비율을 banks.js s1.rows에 반영
```

**주의:** xlsx의 `행정동` 명칭이 주소의 법정동과 다를 수 있습니다 (예: `방배1동` vs `방배동`). 일치하지 않으면 같은 `시군구` 내 유사 행정동명을 검색하거나 팀에서 매핑표를 보완하세요.

## 지역별은행_더미데이터_전체.xlsx 구조

`data/지역별은행_더미데이터_전체.xlsx`는 전국 은행 지점 **더미 거래·수요 데이터**입니다. `계산식_설명` 시트의 Sakasegawa(G/G/c) 공식으로 λ·μ·대기시간이 **이미 계산된 상태**입니다.

### 시트 구성

| 시트명 | 용도 |
|--------|------|
| `은행별_더미데이터` | **메인** — 지점별 이용객·λ·서비스시간·ATM 지표 (banks.js 2~4번 데이터 소스) |
| `계산식_설명` | 더미데이터·λ·μ·창구수 산출 로직 (참고용, **프롬프트 3 불필요 근거**) |

### `은행별_더미데이터` 시트 → banks.js 매핑

| xlsx 컬럼 | banks.js 연동 |
|-----------|---------------|
| `은행명` (또는 주소) | 지점 매칭 키 — `pairKey`와 대조 |
| `일평균 이용객(대표)` | → `s1.rows` **'일평균 방문 고객'** |
| `일평균 이용객(창구)` | → `s1.rows` **'월 창구 거래 건수'** (×22 영업일 월환산, 천 단위 콤마) |
| `고령인구비율` | → `s2.ages` 산출 참고 (60대+ 비중) |
| `ATM 이용비율` | → `s2.channels` 산출 참고 (자동화기기 비중) |
| `시간당 고객도착률 λ(ATM)` | → `s3.lambda` |
| `ATM 평균처리시간(분)` | → `s3.svc` |

### `계산식_설명` 시트 핵심 (프롬프트 3 생략 이유)

- **λ(대표)** = 일평균 이용객(대표) ÷ 7 (영업시간 09:00~16:00 가정)
- **μ(대표)** = 60 ÷ 평균 서비스시간(분)
- **대기시간** = Sakasegawa(G/G/c) 공식으로 산출
- 위 값들이 `은행별_더미데이터` 시트 컬럼에 **이미 채워져 있음** → `s3.lambda`/`s3.svc`를 별도 CSV에서 가져올 필요 없음

### 지점 매칭 키 (데모 4곳)

| banks.js ID | pairKey | xlsx `은행명` 검색 |
|-------------|---------|-------------------|
| hana | 하나은행 강남역금융센터지점 | `하나은행 강남역금융센터지점` |
| kb | KB국민은행 노원종합금융센터 | `KB국민은행 노원종합금융센터` |
| shinhan | 신한은행 여의도중앙금융센터 | `신한은행 여의도중앙금융센터` |
| woori | 우리은행 청담중앙지점 | `우리은행 청담중앙지점` |

#### 📌 프롬프트 0 — 반경 500m ATM/ITM 수 (지금 당장 가능)

> `data/banks/` CSV는 위치 데이터만 있으므로, 인구·거래 없이 **지금 할 수 있는 자동 계산**입니다.

```
data/banks/ 폴더에 지역별 은행 위치 CSV가 있어.
컬럼: place_name, address_name, road_address_name, x, y

banks.js의 각 은행(hana, kb, shinhan, woori)에서
s1.rows 안 '반경 500m 내 ATM/ITM' 행의 값만 계산해서 교체해줘.

계산 방법:
  1. 각 은행의 pairKey에 해당하는 지점 좌표를 CSV에서 찾기
     (서울 지점은 data/banks/서울_banks_최종.csv 참고)
  2. 같은 CSV 전체에서 해당 좌표 반경 500m 이내 ATM/ITM/365 항목 수 세기
  3. s1.rows에서 '반경 500m 내 ATM/ITM' 행의 가운데 값(r[1])만 교체

지점명 매핑:
  hana    → 하나은행 강남역금융센터지점
  kb      → KB국민은행 노원종합금융센터
  shinhan → 신한은행 여의도중앙금융센터
  woori   → 우리은행 청담중앙지점

pairKey, '가장 가까운 대체 점포' 행, s1.score, s1.risks, s2, s3, verdict 등
나머지는 절대 건드리지 마.
data/pairs.js도 건드리지 마.
```

#### 📌 프롬프트 1 — 지점별 인구·고령자 비율 (행정동 xlsx)

> `data/행정동_통합비율_취약점수_최종.xlsx` 시트 `행정동별_비율` 사용. 지점 주소 → 행정동 매핑 후 조회.

```
data/ 폴더에 행정동_통합비율_취약점수_최종.xlsx 파일이 있어.
시트 '행정동별_비율'을 사용해.

각 은행 지점의 주소는 data/pairs.js의 pairKey → 주소 필드에서 가져오고,
법정동명을 추출한 뒤 xlsx에서 시도+시군구+행정동으로 일치하는 행을 찾아줘.

banks.js의 각 은행(hana, kb, shinhan, woori)에서
s1.rows 안의 다음 두 행의 가운데 값(r[1])만 xlsx 실제 값으로 교체해줘:
  - '지역 인구' ← xlsx '총인구수' (단위 '명' 유지, 천 단위 콤마 형식 예: '52,000')
  - '65세 이상 고령자 비율' ← xlsx '고령인구비율' (단위 '%' 유지)

지점 → 행정동 매핑 (조회 키):
  hana    → pairKey '하나은행 강남역금융센터지점' → 서울특별시 / 서초구 / 서초동
  kb      → pairKey 'KB국민은행 노원종합금융센터' → 서울특별시 / 노원구 / 상계동
  shinhan → pairKey '신한은행 여의도중앙금융센터' → 서울특별시 / 영등포구 / 여의도동
  woori   → pairKey '우리은행 청담중앙지점' → 서울특별시 / 강남구 / 청담동

xlsx의 '취약점수(A)' 등 다른 컬럼은 banks.js에 넣지 마.
pairKey는 절대 바꾸지 마.
'가장 가까운 대체 점포' 행은 script.js가 pairKey로 자동 계산하니 건드리지 마.
나머지 구조(s1.score, s2, s3, verdict 등)도 절대 건드리지 마.
data/pairs.js 파일 자체는 수정하지 마.
```

#### 📌 프롬프트 2 — 방문객·거래·연령·채널·λ·svc (더미데이터 xlsx)

> `지역별은행_더미데이터_전체.xlsx` 시트 `은행별_더미데이터` 사용. `계산식_설명` 시트에 따라 λ·svc는 **이미 계산된 컬럼**을 그대로 사용합니다. (구 프롬프트 3 생략)

```
data/ 폴더에 지역별은행_더미데이터_전체.xlsx 파일이 있어.
시트 '은행별_더미데이터'를 사용해.

banks.js의 각 은행(hana, kb, shinhan, woori)에서
아래 값만 xlsx 실제 데이터로 교체해줘:

s1.rows 안:
  - '일평균 방문 고객' ← xlsx '일평균 이용객(대표)' (정수 반올림)
  - '월 창구 거래 건수' ← xlsx '일평균 이용객(창구)' × 22 (영업일 월환산, 천 단위 콤마 형식)

s2:
  - ages: [20~30대%, 40~50대%, 60대이상%] (합계 100)
    · 60대이상% = xlsx '고령인구비율' (퍼센트 값 그대로 사용)
    · 나머지를 20~30대 40% / 40~50대 60% 비율로 배분
  - channels: [모바일%, 자동화기기%, 대면창구%] (합계 100)
    · 자동화기기% = xlsx 'ATM 이용비율' × 100
    · 대면창구% = round(일평균 이용객(창구) / 일평균 이용객(대표) × 100, 1) (최대 50)
    · 모바일% = 100 - 자동화기기% - 대면창구%

s3:
  - lambda ← xlsx '시간당 고객도착률 λ(ATM)' (소수 첫째 자리 반올림)
  - svc ← xlsx 'ATM 평균처리시간(분)' (소수 첫째 자리까지)

지점 매핑 (xlsx '은행명' 열로 검색):
  hana    → 하나은행 강남역금융센터지점
  kb      → KB국민은행 노원종합금융센터
  shinhan → 신한은행 여의도중앙금융센터
  woori   → 우리은행 청담중앙지점

pairKey, '가장 가까운 대체 점포' 행, s1.score, s1.risks, verdict 등 나머지는 절대 건드리지 마.
data/pairs.js 파일 자체는 수정하지 마.
```

#### 📌 프롬프트 3 — 영향도 점수·판정 자동 계산

> 프롬프트 1~2 데이터 입력 후 실행

```
banks.js에 이미 들어간 실제 데이터를 바탕으로
각 은행(hana, kb, shinhan, woori)별로 아래만 계산해서 업데이트해줘.

s1.score: 폐쇄 영향도 점수 (0~100)
  - 고령자 비율 30% 이상 → +30점
  - 고령자 비율 20~30%  → +20점
  - 고령자 비율 10~20%  → +10점
  - 최근접 대체 점포 5km 이상  → +30점 (data/pairs.js의 pairKey → 인근[0].거리 사용)
  - 최근접 대체 점포 2~5km    → +20점
  - 최근접 대체 점포 1km 미만  → +0점
  - 월 창구 거래 7000건 이상  → +20점
  - 월 창구 거래 4000~7000건 → +10점
  - 월 창구 거래 4000건 미만  → +0점
  (합산 후 100 초과 시 100으로 클램프)

s1.risks 4개 항목 value(0~100)와 rating:
  - '고령 고객 의존': 고령자 비율 × 2.5
  - '대체 접근성': 최근접 거리(km) × 10 (최대 100)
  - '거래량 규모': 월 창구 거래 / 100 (최대 100)
  - '디지털 대체성': 100 - 채널_대면%

color: value 70+ → '#C0392B', 40~70 → '#B26A00', 40미만 → '#0E7C6B'

hold: score 75 이상 → true
ovClass: 75+ → 'risk', 40~75 → 'warn', 40미만 → 'ok'

pairKey, s1.rows 지표값, s2.ages/channels, s3.lambda/svc는 바꾸지 마.
data/pairs.js도 건드리지 마.
```

## DATA_GUIDE vs 실제 데이터 차이 (팀 공유용)

| 기존 가정 | 실제 상황 |

## 주의사항

1. **`pairKey`는 절대 바꾸지 마세요.** `data/pairs.js`의 실제 지점명과 연결된 키입니다.

2. **`data/pairs.js`는 건드리지 마세요.** 이미 완성된 파일입니다.

3. **`s3.lambda`와 `s3.svc`는 대기행렬 계산에 직접 사용됩니다.**
   - `지역별은행_더미데이터_전체.xlsx`의 `시간당 고객도착률 λ(ATM)`, `ATM 평균처리시간(분)` 컬럼 사용 (프롬프트 2)
   - 별도 `수요_예측.csv`나 λ·svc 전용 프롬프트는 **불필요** (프롬프트 2에 포함)

4. **`'가장 가까운 대체 점포'` 행은 수동 수정 불필요** — `script.js`가 `pairKey`로 자동 계산합니다.

5. 수정 후 `index.html`을 브라우저에서 새로고침하면 바로 반영됩니다.
