# 데이터 연동 가이드

> 이 문서는 CSV 데이터를 받아 DSS 서비스에 연동하는 방법을 설명합니다.
> **Cursor에서 이 파일을 열어둔 채로 작업**하면 됩니다.

---

## 프로젝트 파일 구조

```
bank-dss/
  index.html          # 화면 (건드리지 않아도 됨)
  style.css           # 디자인 (건드리지 않아도 됨)
  script.js           # 계산 로직 (건드리지 않아도 됨)
  banks.js            # ★ 은행별 지점 데이터 — 여기를 채워야 함
  data/
    pairs.js          # 완료 — 10km 이내 대체 시설 데이터 (건드리지 않아도 됨)
  docs/
    DATA_GUIDE.md     # 이 문서
```

---

## 현재 연동 상태

| 번호 | 데이터 | 연동 상태 | 담당 파일 |
|------|--------|-----------|-----------|
| 5 | 10km 이내 대체 시설 목록 | ✅ 완료 | `data/pairs.js` |
| 1 | 지점별 인구·고령자 비율 | ❌ 대기 중 | `banks.js` |
| 2 | 지점별 방문객·거래·연령·채널 | ❌ 대기 중 | `banks.js` |
| 3 | 피크 시간당 고객 수 (λ) | ❌ 대기 중 | `banks.js` |
| 4 | ITM 1건 처리 시간 | ❌ 대기 중 | `banks.js` |

## CSV 데이터를 받으면 할 일

### STEP 1 — CSV 파일을 프로젝트에 넣기

받은 CSV 파일을 `data/` 폴더 안에 넣어주세요.

```
data/
  pairs.js          # 기존 (건드리지 말 것)
  지점_기본정보.csv  # 1번 데이터
  지점_거래현황.csv  # 2번 데이터
  수요_예측.csv      # 3~4번 데이터
```

---

### STEP 2 — Cursor에서 아래 프롬프트 사용하기

CSV 파일을 `data/` 폴더에 넣고, Cursor 채팅창에 아래 프롬프트를 그대로 붙여넣으세요.

---

#### 📌 1번 데이터: 지점별 인구·고령자 비율

> CSV 컬럼 예시: `지점명, 지역인구, 고령자비율(%)`

```
data/ 폴더에 [파일명.csv] 파일이 있어.
banks.js의 각 은행(hana, kb, shinhan, woori)에서
s1.rows 안의 다음 두 행을 이 CSV의 실제 값으로 교체해줘:
  - '지역 인구' 행의 값
  - '65세 이상 고령자 비율' 행의 값

지점명 매핑:
  hana    → [CSV의 지점명]
  kb      → [CSV의 지점명]
  shinhan → [CSV의 지점명]
  woori   → [CSV의 지점명]

값만 바꾸고 나머지 구조는 절대 건드리지 마.
```

---

#### 📌 2번 데이터: 방문객·거래 건수·연령 분포·채널 이용률

> CSV 컬럼 예시: `지점명, 일평균방문, 월창구거래, 연령20대, 연령40대, 연령60대이상, 채널모바일, 채널기기, 채널대면`

```
data/ 폴더에 [파일명.csv] 파일이 있어.
banks.js의 각 은행(hana, kb, shinhan, woori)에서
아래 값들을 CSV 실제 데이터로 교체해줘:

s1.rows 안:
  - '일평균 방문 고객' 행의 값
  - '월 창구 거래 건수' 행의 값

s2:
  - ages: [20~30대%, 40~50대%, 60대이상%]  (합계 100이 되도록)
  - channels: [모바일%, 자동화기기%, 대면창구%]  (합계 100이 되도록)

지점명 매핑:
  hana    → [CSV의 지점명]
  kb      → [CSV의 지점명]
  shinhan → [CSV의 지점명]
  woori   → [CSV의 지점명]

값만 바꾸고 나머지 구조는 절대 건드리지 마.
```

---

#### 📌 3~4번 데이터: 피크 고객 수 · 처리 시간

> CSV 컬럼 예시: `지점명, 피크시간당고객수, ITM처리시간(분)`

```
data/ 폴더에 [파일명.csv] 파일이 있어.
banks.js의 각 은행(hana, kb, shinhan, woori)에서
s3 안의 값을 CSV 실제 데이터로 교체해줘:
  - lambda: 피크 시간당 예상 고객 수 (숫자)
  - svc: ITM 1건당 평균 처리 시간 (분, 소수 가능)

지점명 매핑:
  hana    → [CSV의 지점명]
  kb      → [CSV의 지점명]
  shinhan → [CSV의 지점명]
  woori   → [CSV의 지점명]

값만 바꾸고 나머지 구조는 절대 건드리지 마.
```

### STEP 3 — 영향도 점수와 판정 업데이트

데이터가 들어오면 `banks.js`의 아래 값도 실제 데이터 기반으로 수정해야 합니다.

```
data/ 폴더에 있는 데이터를 바탕으로 banks.js의 각 은행별로
아래 값을 계산해서 업데이트해줘:

s1.score: 폐쇄 영향도 점수 (0~100)
  계산 기준:
    - 고령자 비율 30% 이상 → +30점
    - 고령자 비율 20~30%  → +20점
    - 고령자 비율 10~20%  → +10점
    - 최근접 대체 점포 5km 이상  → +30점
    - 최근접 대체 점포 2~5km    → +20점
    - 최근접 대체 점포 1km 미만  → +0점
    - 월 창구 거래 7000건 이상  → +20점
    - 월 창구 거래 4000~7000건 → +10점
    - 월 창구 거래 4000건 미만  → +0점
    (합산 후 100 초과 시 100으로 클램프)

s1.risks 4개 항목의 value(0~100)와 rating 텍스트:
    - '고령 고객 의존': 고령자 비율 × 2.5
    - '대체 접근성': 최근접 거리 × 10 (최대 100)
    - '거래량 규모': 월 창구 거래 / 100 (최대 100)
    - '디지털 대체성': 100 - 채널_대면%

color 기준:
    value 70 이상 → '#C0392B' (빨강)
    value 40~70  → '#B26A00' (노랑)
    value 40 미만 → '#0E7C6B' (초록)

hold(전환 보류 여부):
    score 75 이상이면 true, 아니면 false

ovClass:
    score 75 이상 → 'risk'
    score 40~75  → 'warn'
    score 40 미만 → 'ok'

값만 바꾸고 나머지 구조는 절대 건드리지 마.
```

## banks.js 구조 설명

Cursor가 참고할 수 있도록 현재 `banks.js`의 구조를 설명합니다.

```js
const BANKS = {
  은행ID: {                        // hana / kb / shinhan / woori
    name: '표시 이름',             // 드롭다운에 나오는 이름
    branch: '지점명',              // 헤더에 표시되는 지점명
    pairKey: 'PAIRS의 실제 지점명', // data/pairs.js 키 — 건드리지 말 것

    // 종합 판정 (STEP 3 데이터 들어오면 자동 계산으로 대체 예정)
    verdict: '판정 문구',
    hold: false,                   // true면 전환 보류 (ITM 대수 미표시)
    ovClass: 'ok',                 // 'ok'(초록) / 'warn'(노랑) / 'risk'(빨강)
    ovTitle: '개요 배너 제목',      // __REC__: 권고 대수, __TGT__: 목표 대기시간 자동 치환
    ovBody: '개요 배너 본문',

    s1: {                          // STEP 1 사전영향평가
      rows: [
        ['지표명', '값', '단위'],   // 표에 표시되는 6개 행
        // '가장 가까운 대체 점포' 행은 pairKey로 자동 계산됨
      ],
      score: 62,                   // 영향도 점수 0~100 (높을수록 폐쇄 영향 큼)
      risks: [
        ['위험요인명', 막대값0~100, '색상hex', '라벨'],
      ],
      note: '해석 문장 (<b>태그 사용 가능</b>)',
    },

    s2: {                          // STEP 2 디지털 전환 적합성
      ages: [20대%, 40대%, 60대%],  // 합계 100
      channels: [모바일%, 기기%, 대면%], // 합계 100
      grade: 'A',                  // 'A'(적합) / 'B'(조건부) / 'C'(부적합)
      gradeLabel: '적합',
      note: '설명 문장',
    },

    s3: {                          // STEP 3 수요 가정 → 대기행렬 계산 입력값
      lambda: 25,                  // 피크 시간당 고객 수
      svc: 4,                      // ITM 1건당 처리 시간 (분)
    },
  },
};
```

## 주의사항

1. **`pairKey`는 절대 바꾸지 마세요.** `data/pairs.js`의 실제 지점명과 연결된 키입니다. 바꾸면 인근 시설 목록과 최근접 거리가 깨집니다.

2. **`data/pairs.js`는 건드리지 마세요.** 이미 완성된 파일입니다.

3. **`s3.lambda`와 `s3.svc`는 STEP 3·4 대기행렬 계산에 직접 사용됩니다.** 값이 이상하면 권고 대수와 시뮬레이션이 엉뚱하게 나올 수 있습니다.
   - `lambda`는 붐비는 시간대 기준으로 잡으세요 (일평균이 아닌 피크 기준).
   - `svc`는 보통 3~8분 사이입니다.

4. **Cursor 프롬프트 안의 `[파일명.csv]`, `[CSV의 지점명]`은 실제 값으로 바꿔서 사용**하세요.

5. 수정 후 `index.html`을 브라우저에서 새로고침하면 바로 반영됩니다.
